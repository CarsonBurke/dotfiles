# Add Skill

- Create a skill at `~/Documents/repositories/dotfiles/skills` with frontmatter (name, description) and an instruction body.
- Be consistent with and get insight from existing skills in the directory.
- Write the skill for agents in mind.
- Write for a version of yourself with no memory of this session: assume the future agent is smart and mostly needs durable context, intent, and judgment boundaries.
- Think about the role, judgment, and task the skill is for, then write instructions that help the agent make good decisions in varied real contexts.
- Prefer intent, principles, boundaries, and expected output over rigid procedures.
- Generally avoid exact step-by-step workflow unless essential for correctness, safety, or interoperability.
- Give the agent room to adapt scope, tools, delegation, and level of detail to the situation.
- Describe what high-quality execution looks like, including tradeoffs.
